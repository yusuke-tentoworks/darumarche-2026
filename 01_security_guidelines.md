# セキュリティ対策ガイドライン：AntigravityによるWeb開発（初心者・非エンジニア向け）

このドキュメントは、コーディングの知識が少ないユーザーが Antigravity（AI）を活用してWeb制作を行う際、自分と顧客を守るための「確実に守るべきルール」をまとめたものです。

---

## 🛑 【最重要】絶対にやってはいけないこと（ゴールデンルール）
以下の3点は、いかなる場合も例外なく守ってください。

1. **パスワード・秘密鍵をチャットに貼らない**: 
   - 「このAPIキーで動くようにして」とチャット欄に実際の値を貼り付けるのは厳禁です。
2. **本番データをそのまま使わない**: 
   - 顧客のメールアドレスリストや個人情報をAIに読み込ませてはいけません。必ず「サンプルデータ」を作成して依頼してください。
3. **「.env」をGitHubに上げない**: 
   - 鍵がかかっていない家と同じになります。`.gitignore` が正しく設定されているか、AIに確認させてください。

---

## 1. Antigravity（AI）とのやり取りで行う対策

コーディングの知識がなくても、AIに「命令」することでセキュリティを担保します。

### 1-1. セキュリティ監査の自動依頼
タスクが完了したら、必ず以下のプロンプトを Antigravity に送ってください。
> **「今作成したコードに、XSS、SQLインジェクション、機密情報の露出などのセキュリティ脆弱性がないか、最新の脅威（OWASP Top 10など）に照らして診断し、修正が必要な場合は提案してください。」**

### 1-2. 依存ライブラリのチェック
AIが提案した新しいパッケージを使う際は、以下を確認させてください。
- 「このライブラリは現在もメンテナンスされており、安全なものですか？」
- 「より有名で安全な代替案はありますか？」

---

## 2. GitHub と 環境設定の対策

### 2-1. .env（秘密のメモ帳）の管理
- 機密情報はすべて `.env` というファイルに書きます。
- このファイルは「Gitにコミット（保存）しない」設定が必須です。
- **確認方法**: GitHubのウェブサイトで自分のリポジトリを見た際、`.env` というファイルが表示されていたら、即座に削除とパスワード変更が必要です。

### 2-2. 2段階認証 (2FA) の有効化
- GitHubアカウントには必ず2段階認証を設定してください。これにより、パスワードが漏れても不正ログインを防げます。

### 2-3. GitHub Actions (自動化工程) での秘密情報の扱い
GitHub Actions（サイトマップの自動更新や自動デプロイなど）で APIキーが必要な場合、コード内に直接書くことは厳禁です。

- **GitHub Secrets の利用**: リポジトリの `Settings > Secrets and variables > Actions` から「New repository secret」として登録します。
- **ワークフローファイル (`.yml`) への設定**: 登録しただけではプログラムから読み込めません。`.github/workflows/` 内のファイルで、`env:` セクションを使用してシークレットをプログラムに渡す設定が必要です。
- **エラー時の確認**: 「Process completed with exit code 1」や「API_KEY is not set」というエラーが出た場合は、この設定漏れを疑ってください。

---

## 3. インフラと公開後の対策（コードの不備を補う）

コードに知識不足による隙があったとしても、外部のサービスでガードを固めます。

- **Cloudflare (WAF/CDN) の導入**: 
    - サイトの前に「盾」を置くイメージです。攻撃的な通信（ボットや海外からの不審なアクセス）を自動で遮断してくれます。
- **SSL（https）の必須化**: 
    - ブラウザの「保護されていない通信」という警告が出ないよう、必ずSSL化（通信の暗号化）を行います。

## 4. セキュリティチェックリスト（公開前必須）
公開前に、以下のチェックを済ませたか AI に聞いてください。

- [ ] APIキーやパスワードがファイル内に直書き（ハードコード）されていないか？
- [ ] ユーザーが入力するフォームに、変なプログラムを埋め込む攻撃（XSS）対策はされているか？
- [ ] エラーメッセージに、サーバーの内部構造やパスワードが含まれていないか？
- [ ] `.aiignore` と `.gitignore` は正しく機能しているか？

---

## 5. 万が一「やらかした」と思ったら
1. **パスワード・APIキーの無効化**: 漏洩した疑いがある鍵を、すぐに発行元（AWSやGoogle Cloud等）で消去または再発行してください。
2. **履歴の強制書き換え**: 自分で行うのが難しい場合は、Antigravityに「GitHubから特定のファイルを履歴ごと安全に消去するコマンドを教えて」と聞き、指示に従ってください。

## 付録：環境別・標準セキュリティ実装パターン

**APIキーをブラウザから隠す（サーバーサイド・プロキシ）**を実装する際、利用するサーバー環境によって以下の最適な方法を選択してください。

### 1. エックスサーバー / 一般的なレンタルサーバー
- **方式**: **PHPプロキシ方式**
- **仕組み**: `main.js`（表側）から直接外部APIを叩かず、サーバー内に作成した PHPファイル を経由してデータを取得します。
- **指示のヒント**: 「PHPを使って、APIキーをサーバーサイドで隠蔽して通信するプロキシを作成してください」と依頼します。

### 2. Netlify
- **方式**: **Netlify Functions + 環境変数**
- **仕組み**: Netlifyが提供する「Functions」機能を利用し、APIキーは管理画面（Environment Variables）に保存します。ファイルとして保存しないため、最も安全です。
- **指示のヒント**: 「Netlify Functionsを使ってAPIキーを隠蔽してください。キーは管理画面の環境変数から読み込むようにしてください」と依頼します。
